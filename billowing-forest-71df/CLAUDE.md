## Commands

```bash
npm run dev        # Start local dev server via wrangler
npm run deploy     # Deploy to Cloudflare Workers (minified)
npm run cf-typegen # Regenerate worker-configuration.d.ts from wrangler config
npm run typecheck  # TypeScript check without emitting
```

## Architecture

This is a **Hono** web application running on **Cloudflare Workers**.

- `src/index.ts` — single entry point; exports a Hono app instance as the default export (required by Workers runtime)
- `public/` — static assets served via Cloudflare Assets binding (`ASSETS`); `index.html` fetches `/message` on load
- `worker-configuration.d.ts` — auto-generated by `cf-typegen`; defines `CloudflareBindings` (currently only `ASSETS: Fetcher`)
- `wrangler.jsonc` — Worker config: name, entrypoint, assets binding, compatibility flags (`nodejs_compat`)

### Key patterns

- Type Cloudflare bindings via `new Hono<{ Bindings: CloudflareBindings }>()` — regenerate types after changing `wrangler.jsonc` bindings
- JSX is configured with `hono/jsx` as the import source (available if needed)
- Global middleware: `logger()` and `secureHeaders()` applied to all routes
- Error handling: `app.notFound` and `app.onError` handlers return JSON
